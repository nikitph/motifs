# Systems Innovation Compiler (SIC) — LLM Skill

## What This Is

The Systems Innovation Compiler is a structured methodology for diagnosing, analyzing, and innovating complex systems. It combines five frameworks into a single analytical engine:

- **32 Motifs** — substrate-independent structural concerns (State, Boundary, Feedback, Authority, etc.)
- **205 Primitives** — irreducible operations across 20 computational layers, from physics to autonomous agents
- **106 Innovation Methods** — systematic transformations for generating, validating, and composing novel designs
- **290 Distortions** — how every structural concern gets bent, with composition grammar and remediation
- **Institution Motifs** — human coordination patterns (courts, registrars, archives) as pre-validated design templates

## How to Use

1. **Copy the entire contents of `sic_skill.md`**
2. **Paste it into your LLM's system prompt** (or custom instructions, or project knowledge)
3. **Ask it to analyze any system, technology, or problem**

## Example Prompts

- *"Apply the SIC pipeline to analyze Kubernetes"*
- *"Diagnose the motif signature of modern email systems and find innovation opportunities"*
- *"What's the load-bearing assumption in current recommendation systems? Apply M3 and the Innovation Engine"*
- *"Run a distortion analysis on social media feed algorithms — which primitive dynamics are active?"*
- *"Use the density matrix approach to find empty cells in autonomous vehicle systems"*
- *"Apply Convergent Transfer to this system design — what mature system has a structurally similar signature?"*
- *"Ground this microservice architecture in institutional precedent — what offices and bypass paths exist?"*

## What It Does Well

- **Structural diagnosis**: Identifies absent, impoverished, or distorted motifs in any system
- **Cross-domain transfer**: Finds solutions from distant domains by matching motif signatures
- **Innovation generation**: Systematically produces typed, validated, cost-classified candidates
- **Distortion prediction**: Predicts how a system will fail under adversarial pressure before deployment
- **Institutional grounding**: Provides centuries of adversarially tested coordination patterns for governance concerns

## Works With

Any LLM that accepts system prompts or custom instructions: Claude, GPT-4, Gemini, Llama, Mistral, etc. The skill is model-agnostic.

## Size

~4,500 words / ~6,500 tokens. Fits comfortably in any modern LLM's context window alongside substantial problem descriptions.
